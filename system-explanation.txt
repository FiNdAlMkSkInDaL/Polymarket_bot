SECTION 1: SYSTEM OVERVIEW & TRADING PHILOSOPHY
================================================================================

What it is:
  A fully automated high-frequency trading bot for Polymarket binary prediction
  markets. It runs on Polygon (chain ID 137) and places orders on the
  Polymarket CLOB (Central Limit Order Book).

Core strategy:
  CONTRARIAN MEAN-REVERSION on the NO token.

  When the YES token experiences a sudden price spike (retail panic buying),
  the bot buys the NO token at a temporary discount, betting that the spike
  will revert. The NO token is always cheaper during YES panic because
  YES + NO = $1 in a binary market.

  Secondary strategies: drift signal (slow-bleed displacement in low-vol
  regimes), RPE (model-based probability divergence for crypto markets),
  cross-market stat-arb, and spread-compression entries.

How money is made:
  1. YES spikes from 0.60 to 0.68 on retail panic
  2. NO drops correspondingly from 0.40 to 0.32
  3. Bot buys NO at 0.32 (entry)
  4. Spike reverts, NO returns to ~0.38 (exit at take-profit)
  5. Profit = ~6 cents per share (minus fees)

Market:
  - Polymarket CLOB: REST + WebSocket, binary YES/NO markets
  - Assets: YES and NO tokens per condition (market question)
  - Price range: 0.01 to 0.99 (probabilities)
  - Tick size: 1 cent ($0.01)
  - Fee model: Dynamic parabolic curve — f(p) = f_max × 4 × p × (1 − p)
    Peak at p=0.50 (1.56%), zero at tails. Maker fee = 0 bps.
  - Chain: Polygon mainnet (137), settlements on-chain

Deployment modes:
  PAPER       — simulated fills, mocked $50 wallet, data recording forced ON
  PENNY_LIVE  — real CLOB + real wallet, HARDCODED $1 max per trade (un-overridable)
  PRODUCTION  — all guardrails lifted, requires --confirm-production flag

VPS:
  Runs on a Linux VPS (135.181.85.32) as a systemd service (polymarket-bot.service).
  Secrets stored in /dev/shm/secrets/.env (tmpfs, never touches disk).

SECTION 3: CORE ARCHITECTURE (bot.py)
================================================================================

Entry point: `polybot run` (cli.py) → TradingBot(deployment_env).start()

The TradingBot class is the central orchestrator. It owns ~30 subsystem
instances and runs 18+ concurrent asyncio tasks.

INITIALIZATION SEQUENCE:
  1. Resolve deployment environment (PAPER/PENNY_LIVE/PRODUCTION)
  2. Create DeploymentGuard (phase-aware size caps)
  3. Create OrderExecutor (paper or live CLOB client)
  4. Create TradeStore (SQLite)
  5. Create PortfolioCorrelationEngine (Pillar 15)
  6. Create PositionManager (receives executor, store, guard, PCE, book_trackers)
  7. Create WhaleMonitor (Polygonscan polling)
  8. Create TelegramAlerter
  9. Create MarketLifecycleManager
  10. Create LatencyGuard
  11. boot_start() → setup_logging → credential gate → _run()

_run() SEQUENCE:
  1. Initialize TradeStore (SQLite schema)
  2. Discover markets → wire each market (create aggregators, detectors, books)
  3. Set wallet balance
  4. Fee model validation (live only)
  5. Create WebSocket connections (trade + L2)
  6. Create AdverseSelectionGuard
  7. Create BookHeartbeat
  8. Create OrderStatusPoller
  9. Create StopLossMonitor
  10. Create AdverseSelectionMonitor (maker fill tracking)
  11. Launch 18+ asyncio tasks
  12. Install signal handlers
  13. await asyncio.gather(*tasks)

MARKET WIRING (_wire_market):
  For each market, creates:
  - OHLCVAggregator (YES) + OHLCVAggregator (NO)
  - PanicDetector (wired to both aggregators)
  - CompositeSignalEvaluator (imbalance + spread compression signals)
  - RegimeDetector (SI-1, if enabled)
  - MeanReversionDrift (V3, if enabled)
  - L2OrderBook per token (with BBO callback and iceberg detector)
  - L2OrderBookAdapter (wraps L2 behind OrderbookTracker interface)
  - PCE market registration

CONCURRENT TASKS (18+):
  ws                  — Trade/book WebSocket
  trade_processor     — Main trade consumption loop (signals fire here)
  book_processor      — Orderbook update consumption
  timeout_loop        — Position timeout checks (10s)
  stats_loop          — Stats broadcast (15 min)
  whale_monitor       — Polygonscan polling
  market_refresh      — Re-discovery/re-scoring (10 min)
  cleanup             — Stale cleanup + daily PnL reset (5 min)
  tp_rescale          — Adaptive TP target rescaling (15s)
  adverse_sel         — Fast-kill detection (50ms)
  heartbeat           — Book staleness detection
  ghost_liquidity     — Ghost liquidity circuit breaker (500ms)
  order_status_poller — CLOB order fill polling (2s)
  health_reporter     — system_health.json writer (60s)
  rpe_retrigger       — BTC spot retrigger (5s)
  pce_refresh         — Correlation matrix recalc (30 min) + prior validation (every 6h)
  stale_bar_flush     — Flush stale OHLCV bars (30s)
  paper_summary       — Paper trade summary (30 min)
  data_recorder       — (conditional) Data recording
  l2_ws              — (conditional) L2 WebSocket

THE THREE SIGNAL PATHS:

  Path 1: PanicDetector (primary)
    YES bar close → guards → PanicDetector.evaluate() → PanicSignal
    → EQS gate → PositionManager.open_position() → entry chaser

  Path 2: DriftSignal (V3, secondary)
    Only fires when PanicDetector is SILENT (signals are uncorrelated).
    YES bar close → drift cooldown → MeanReversionDrift.evaluate() → DriftSignal
    → synthesize PanicSignal → PositionManager.open_position()

  Path 3: RPE (Pillar 14, independent)
    YES bar close → data freshness → RPE.evaluate() → SignalResult
    → EQS gate → calibration tracking → PositionManager.open_rpe_position()

ORDER EXECUTION FLOW:
  Signal fires → open_position() → risk gates (6 checks) → entry price (ask-1¢)
  → EQS scoring → Kelly sizing → place BUY limit order → OrderChaser
  → POST_ONLY repricing loop → fill → place SELL at TP → exit chaser → close

GRACEFUL SHUTDOWN:
  stop(): _running=False → cancel_all orders → cancel chasers → stop all
  subsystems → final stats → trade_store.clear_live_state() → close()

SECTION 6: DATA LAYER (Market Data Pipeline)
================================================================================

Two parallel WebSocket connections:

  A) MarketWebSocket (src/data/websocket_client.py)
     - Subscribes to "market" channel for all tracked assets
     - Routes: trades → trade_queue, book events → book_queue
     - TradeEvent dataclass: timestamp, market_id, asset_id, side, price, size
     - Auto-reconnect with exponential backoff (1s to 60s) + jitter
     - Silence watchdog: closes WS if no messages for ws_silence_timeout_s
     - Bounded queues: drops OLDEST on overflow (preserves latest state)

  B) L2WebSocket (src/data/l2_websocket.py)
     - Dedicated connection for L2 deltas (separated to prevent delta storms
       from starving trade processing)
     - Routes to L2OrderBook instances directly
     - Handles snapshot fetching on connect/desync
     - Retry with exponential backoff

L2OrderBook (src/data/l2_book.py, ~790 lines):
  CRITICAL COMPONENT — full-depth local order book reconstruction

  State machine: EMPTY → BUFFERING → SYNCED ↔ DESYNCED
  Uses SortedDict from sortedcontainers for O(log n) operations.
  Bids stored with negated keys (ascending iteration = descending price).

  Key properties:
    is_reliable — True if delta_count < 50 OR seq_gap_rate < 2%
    seq_gap_rate — desync_total / delta_count (NEVER reset)
    best_bid, best_ask, spread_cents, book_depth_ratio

  Recovery flow:
    Seq gap detected → DESYNCED → fire on_desync callback →
    L2WebSocket._on_book_desync() → begin_buffering() →
    REST snapshot fetch (up to 3 retries) → apply_snapshot() →
    replay buffered deltas → SYNCED

  depth_near_mid_usd(cents, max_levels):
    Iterates SortedDict keys directly — ZERO Level allocations.
    Designed for the 50ms adverse selection polling loop.

OrderbookTracker (src/data/orderbook.py):
  Simpler top-of-book tracker from WS price_change events.
  L2OrderBookAdapter wraps L2OrderBook behind this interface.
  Also exposes is_reliable (always True), seq_gap_rate (0.0), delta_count (0),
  desync_total (0) as no-op stubs so all consumer code works regardless of
  whether L2 is enabled. Plain trackers have no desync concept, so is_reliable
  is unconditionally True.

OHLCVAggregator (src/data/ohlcv.py):
  Builds 1-minute bars from trade events.

  Key outputs:
    rolling_vwap         — volume-weighted average across lookback window
    rolling_volatility   — std dev of 1-min log returns
    rolling_volatility_30m — 30-minute window for TP rescaling
    rolling_volatility_ewma — EWMA σ (RiskMetrics λ=0.94, fast-reacting)
    avg_bar_volume       — mean bar volume
    last_trade_time      — updated on EVERY trade
    current_price        — last traded price

  flush_stale_bar(now): Closes current bar if open > 60s.
    Critical for low-volume markets to prevent frozen VWAP/σ.

  _BarBuilder: internal, caps at 10,000 ticks per bar to prevent
  unbounded memory growth. On overflow, keeps most recent half.

Market Discovery (src/data/market_discovery.py):
  3-tier fetching: CLOB REST → Gamma events → Gamma markets
  MarketInfo: condition_id, question, yes/no_token_ids, volume, end_date,
    event_id, liquidity, score, accepting_orders, tags

  Filters: active, not closed, accepting orders, binary (2 tokens),
  volume ≥ min, liquidity ≥ min, resolution ≥ min_days

Market Lifecycle (src/data/market_lifecycle.py):
  4-tier state machine: observing → active → draining → emergency

  Market scoring: 7-component weighted composite (0-100):
    volume(0.20), liquidity(0.20), spread(0.20), time_to_resolve(0.15),
    fee_zone(0.10), price_range(0.05), trade_freq(0.10)
    Plus: whale_interest bonus, mti_penalty deduction

PanicDetector (src/signals/panic_detector.py):
  PRIMARY SIGNAL — fires on YES price/volume spikes.
  Returns PanicSignal: market_id, prices, zscore, volume_ratio, no_best_ask, whale

  Gate sequence (all must pass):
    1. History gate: ≥ 5 bars in aggregator
    2. VWAP/σ validity: σ > 0, vwap > 0
    3. Z-score computation: (bar.close - vwap) / σ
    4. Intra-bar retracement discount: if price already reverting within bar
       (close in lower half), discounts z-score by 0.70-0.85
    5. Volume ratio: bar.volume / avg_bar_volume
    6. Z-score gate: zscore ≥ zscore_threshold
    7. Volume ratio gate: v_ratio ≥ volume_ratio_threshold
    8. NO discount check: no_best_ask < no_vwap × no_discount_factor
       (NO must actually be discounted vs its VWAP)
    9. Trend guard: suppress if YES has moved ≥ trend_guard_pct over
       trend_guard_bars (regime shift, not panic)

MeanReversionDrift (src/signals/drift_signal.py):
  V3 SIGNAL — captures slow-bleed displacement in low-vol regimes.
  Intentionally orthogonal to PanicDetector. Only fires when panic is SILENT.
  Returns DriftSignal: displacement, score, direction, ewma_vol

  Gate sequence (all must pass):
    1. Regime: regime_is_mean_revert must be True
    2. L2 reliability: l2_reliable must be True
    3. History: ≥ lookback_bars in NO aggregator
    4. EWMA vol ceiling: vol > 0 AND vol < vol_ceiling (low-vol only)
    5. No high-volume bars: all bars in window have vol/avg ≤ 1.5
       (ensures uncorrelated with PanicDetector)
    6. Displacement threshold: |displacement| ≥ z_threshold

  Displacement: (current_price - vwap) / sigma
  Direction: negative → "BUY_NO" (NO is cheap), positive → "SELL_NO"
  Score: min(1.0, |displacement| / (2 × z_threshold)), saturates at 2x

EdgeFilter / EQS (src/signals/edge_filter.py):
  PRE-TRADE QUALITY GATE — geometric mean of 4 normalised factors:

  EQS = 100 × R^0.35 × F^0.30 × T^0.20 × S^0.15

  Where:
    R = regime_quality = binary_entropy(entry_price)
        Markets near 50/50 → max uncertainty → noise → mean-revertible
    F = fee_efficiency = max(floor, 1 - fee_cents/gross_cents)
        floor = 0.10 prevents fee from zeroing entire geometric mean
    T = tick_viability = min(1.0, discretised_net_cents / 3.0)
        How many 1¢ ticks the expected move spans on Polymarket grid
    S = signal_quality:
        Panic: 0.5 + 0.35(1-exp(-0.5·z_excess)) + 0.20·min(v_excess, 2.0)
        Model: clamp(confidence, 0.1, 1.0)
        Whale bonus: +0.15

  CRITICAL PROPERTY: geometric mean → any factor at zero zeros entire score.

  Regime-adaptive threshold (OE-6):
    When vol is available: ratio = ewma_vol / eqs_vol_ref
    adjustment = clamp(1 - ratio, -0.25, +0.25)
    threshold *= (1 + adjustment)
    → High vol lowers threshold, low vol raises it

  ConfluenceContext: whale_strong_confluence, spread_compressed, l2_reliable,
    regime_mean_revert — 4 binary factors

  compute_confluence_discount(ctx, base_threshold):
    1. HARD GATE: if !l2_reliable → return base (no discounts)
    2. Floor: maker_routing_active → 40.0, else → 35.0 (Flaw 1)
    3. Accumulate discounts for whale (4.0), spread (4.0), l2 (0.0), regime (0.0)
       Regime suppressed for drift signals (Flaw 2)
    4. Need ≥ confluence_min_factors active to apply
    5. Result: max(floor, base - sum(discounts))

RegimeDetector (src/signals/regime_detector.py):
  Per-market HMM-lite two-state model: MEAN_REVERT vs TRENDING
  Score ∈ [0, 1], higher = more mean-revertible
  is_mean_revert = score ≥ 0.40

  Three features combined via geometric mean:
    autocorrelation: lag-1 ACF mapped through sigmoid(-ac1 × 3)
      ac1 = -1 → 1.0 (strong MR), ac1 = +1 → 0.0 (trending)
    vol_ratio: ewma_vol / equal_weight_vol, sigmoid-mapped
      ratio > 2 → trending, ratio < 0.5 → calm/MR
    persistence: fraction of same-sign returns, sigmoid-mapped
      ~0.5 random → MR friendly, > 0.7 → trending

  Score smoothed via EWMA: score = λ·score + (1-λ)·raw

RPE (src/signals/resolution_probability.py):
  Pillar 14 — model-based alpha.

  CryptoPriceModel: log-normal Black-Scholes for "Will BTC exceed $X" markets
    P(S_T > K) = Φ(d2), d2 = (ln(S/K) + (0 − σ²/2)T) / (σ√T)
    Risk-neutral drift (μ=0) — prediction markets are risk-neutral
    Adaptive σ via EWMA of log-return² from spot feed

  GenericBayesianModel: Beta-Binomial model for non-crypto markets
    NOW ACTIVE — Dynamic Prior Generation Engine (DPGE) replaces static Beta(2,2)

    5-step estimate() flow:
      1. Tag prior — TagPriorRegistry.get_prior(tags) → (α₀, β₀, label)
         Falls back to prior_k market-anchored prior when dynamic priors disabled
      2. Time kernel — w_prior = sigmoid(γ × (t − t_half))
         t = days_remaining / total_duration, γ=8.0, t_half=0.15
         High early in market life, decays steeply near expiry
      3. L2 pseudo-counts — Δ = κ_eff × clamp(ln(book_depth_ratio), −2, 2)
         κ_eff = rpe_l2_kappa × w_prior; only applied when l2_reliable=True
         Buy-side pressure (ratio>1) → additive YES pseudo-counts; sell-side → NO
      4. Bayesian update — posterior = (α₀ + obs × w_price + Δ) / (α₀ + β₀ + n_eff + |Δ|)
         n_eff = obs_weight × w_obs / (w_prior + ε), capped at 200
         w_obs = 1 − w_prior (complementary weights sum to 1)
      5. Confidence — (α_post + β_post) / (α₀ + β₀ + n_eff + |Δ| + 1) × w_prior
         Near-expiry suppression: w_prior → 0 → confidence → 0 (terminal noise filter)

    New metadata keys: prior_source, w_prior, n_eff, kappa_eff, l2_active,
      dynamic_prior_enabled, alpha0, beta0

Kelly Sizer (src/trading/sizer.py):

  FULL KELLY:
    1. Win probability: max(0.01, decayed_wr + 0.15 × signal_score), capped at p_cap
    2. Edge discount: p_adj = 0.5 + (p_raw - 0.5) × (1 - uncertainty_penalty)
    3. Payoff ratio: b = |avg_win| / |avg_loss|
    4. Edge: p_adj × b - (1 - p_adj). If ≤ 0 → reject
    5. Kelly fraction: f* = edge / b, f_adj = f* × kelly_fraction (quarter Kelly)
    6. Capital caps: min(f_adj × bankroll, bankroll × max_pct/100, max_trade)

SECTION 14: KEY FORMULAS & ALGORITHMS
================================================================================

EQS (Edge Quality Score):
  EQS = 100 × R^0.35 × F^0.30 × T^0.20 × S^0.15
  R = binary_entropy(p) = -p·log₂(p) - (1-p)·log₂(1-p)
  F = max(0.10, 1 - fee_cents/gross_cents)
  T = min(1.0, discretised_net_cents / 3.0)
  S = 0.5 + 0.35(1-exp(-0.5·z_excess)) + 0.20·min(v_excess, 2.0)

Kelly Criterion:
  f* = (p·b - q) / b, where b = |avg_win/avg_loss|, q = 1-p
  f_adj = f* × 0.25 (quarter Kelly)

Dynamic Fee Curve:
  f(p) = f_max × 4 × p × (1 − p), f_max = 0.0156

Take-Profit:
  target = entry + α × (VWAP - entry)
  α adjusted by: vol(±0.04-0.08), depth(+0.03), whale(+0.08),
    time(-0.05), VWAP proximity(±0.04-0.05)

Dynamic Alpha (Adverse Selection):
  α_dynamic = α_base × (σ_rolling / σ_ref)^γ, clamped [0.01, 0.15]

Welch t-test (one-sided):
  t = x̄ / (s / √n), H₀: μ ≥ 0, H₁: μ < 0
  Pure-Python CDF via Lentz continued-fraction (no scipy)

VaR (Portfolio):
  VaR = z × √(w' Σ w), Σ_ij = σ_i × σ_j × ρ_ij, z = 1.645 (95%)

Regime Score:
  raw = (autocorr × vol_ratio × persistence)^(1/3)  (geometric mean)
  score = λ·score + (1-λ)·raw  (EWMA smoothing)

EWMA Volatility:
  σ²_t = λ·σ²_(t-1) + (1-λ)·r²_t, λ = 0.94 (RiskMetrics)

Crypto RPE (Black-Scholes):
  P(S_T > K) = Φ(d₂)
  d₂ = (ln(S/K) + (0 - σ²/2)T) / (σ√T)  [risk-neutral, μ=0]

DPGE (Dynamic Prior Generation Engine) — GenericBayesianModel:
  Time-decay kernel (sigmoid):
    t = days_remaining / total_duration_days
    w_prior(t) = 1 / (1 + exp(−γ × (t − t_half))),  γ=8.0, t_half=0.15
    → near t=0 (at expiry): w_prior → 0  (prior irrelevant, suppress noise)
    → near t=1 (new market): w_prior → 1  (prior dominates sparse data)

  L2 imbalance pseudo-counts:
    raw = clamp(ln(book_depth_ratio), −2.0, +2.0)
    κ_eff = rpe_l2_kappa × w_prior
    Δ_yes = max(0, raw × κ_eff),  Δ_no = max(0, −raw × κ_eff)

  Bayesian posterior (Beta-Binomial conjugate update):
    n_eff = min(200, obs_weight × w_obs / (w_prior + ε)),  w_obs = 1 − w_prior
    α_post = α₀ + market_price × n_eff + Δ_yes
    β_post = β₀ + (1 − market_price) × n_eff + Δ_no
    probability = α_post / (α_post + β_post)

  Confidence (near-expiry suppression):
    concentration = α_post + β_post
    confidence = (concentration / (α₀ + β₀ + n_eff + Δ_total + 1)) × w_prior

SECTION 15: IMPORTANT DESIGN PATTERNS & GOTCHAS
================================================================================

1. FROZEN DATACLASS CONFIGS
   StrategyParams is frozen. Tests override via object.__setattr__ + try/finally.
   Never modify settings.strategy directly in production code.

2. FIRE-AND-FORGET PATTERN
   _safe_fire_and_forget(coro) wraps in asyncio.ensure_future with
   _safe_task_done_callback to log any unhandled exceptions.
   Used for Telegram notifications (off the hot path).

3. DUAL BOOK SYSTEMS
   L2OrderBook (full depth, SortedDict) vs OrderbookTracker (top-of-book).
   L2OrderBookAdapter bridges L2 behind Tracker interface.
   Both coexist — some code paths use L2, others use Tracker.
   Both types expose is_reliable, seq_gap_rate, delta_count, desync_total:
   plain trackers return safe non-L2 defaults (True, 0.0, 0, 0).
   Consumer code MUST NOT isinstance-check — treat both as duck-typed.

4. FAST_KILL_EVENT
   asyncio.Event shared between AdverseSelectionGuard and all OrderChasers.
   Cleared = pauses all chasers. Set = resume.
   Critical path: 50ms adverse detection → clear event → cancel orders.

5. ENTRY LOCK (asyncio.Lock)
   PositionManager._entry_lock serializes all position entries.
   Prevents per-event burst from opening multiple positions simultaneously.

6. NEGATED BIDS IN SORTEDDICT
   Both L2OrderBook and MatchingEngine store bids with negated prices:
   _bids[−price] = size. This makes SortedDict ascending iteration
   yield prices in descending order (best bid first).

7. SEQUENCE GAP TRACKING
   L2OrderBook.desync_total is NEVER reset (cumulative lifetime counter).
   desync_count resets on each snapshot. seq_gap_rate = total / delta_count.
   is_reliable = gap_rate < 2%. This drives the L2 hard gate decision.

8. MAKER FEE = 0
   Polymarket charges 0 bps for maker (POST_ONLY) orders.
   This is why V1 maker routing eliminates the fee_efficiency factor.
   compute_edge_score with execution_mode="maker" → entry/exit fees = 0.

9. ZERO-ALLOCATION DEPTH QUERY
   L2OrderBook.depth_near_mid_usd iterates SortedDict keys directly.
   No Level objects allocated. Designed for 50ms adverse selection loop.

10. BAR BUILDER TRUNCATION
    OHLCVAggregator._BarBuilder caps at 10,000 ticks per bar.
    On overflow, keeps most recent half but preserves _true_open and
    _total_volume. Prevents unbounded memory in high-frequency markets.

11. ATOMIC FILE WRITES
    system_health.json written via tempfile → os.replace() in a thread.
    CorrelationMatrix.save() uses same pattern.

12. CRASH RECOVERY
    Every 5 minutes: checkpoint_orders + checkpoint_positions to SQLite.
    On restart: restore_orders + restore_positions.
    Clean shutdown: clear_live_state().

13. PROBE EXCLUSION FROM EXPECTANCY
    TradeStore.get_rolling_expectancy filters WHERE is_probe=0.
    Prevents micro-sized probes from polluting the Kelly estimator.

14. BOOTSTRAP FACTOR AUDIT RESULTS
    Whale (p=0.041): justified, weight 5→4
    Spread (p=0.023): justified, unchanged at 4
    L2 (p=0.38): NOT justified, zeroed → hard gate
    Regime (p=0.52): NOT justified, zeroed
    These p-values from scripts/confluence_sensitivity.py with B=2000.

15. THREE FLAW GUARDS
    Flaw 1: Maker + confluence amplification → combined floor 40.0
    Flaw 2: Drift + regime double-count → suppress regime for drift
    Flaw 3: Stale whale in probes → suppress confluence for probes
