SECTION 1: SYSTEM OVERVIEW & TRADING PHILOSOPHY
================================================================================

What it is:
  A fully automated high-frequency trading bot for Polymarket binary prediction
  markets. It runs on Polygon (chain ID 137) and places orders on the
  Polymarket CLOB (Central Limit Order Book).

Core strategy:
  CONTRARIAN MEAN-REVERSION on the NO token.

  When the YES token experiences a sudden price spike (retail panic buying),
  the bot buys the NO token at a temporary discount, betting that the spike
  will revert. The NO token is always cheaper during YES panic because
  YES + NO = $1 in a binary market.

  Secondary strategies: drift signal (slow-bleed displacement in low-vol
  regimes), RPE (model-based probability divergence for crypto markets),
  cross-market stat-arb, and spread-compression entries.

How money is made:
  1. YES spikes from 0.60 to 0.68 on retail panic
  2. NO drops correspondingly from 0.40 to 0.32
  3. Bot buys NO at 0.32 (entry)
  4. Spike reverts, NO returns to ~0.38 (exit at take-profit)
  5. Profit = ~6 cents per share (minus fees)

Market:
  - Polymarket CLOB: REST + WebSocket, binary YES/NO markets
  - Assets: YES and NO tokens per condition (market question)
  - Price range: 0.01 to 0.99 (probabilities)
  - Tick size: 1 cent ($0.01)
  - Fee model: Dynamic parabolic curve — f(p) = f_max × 4 × p × (1 − p)
    Peak at p=0.50 (2.00%), zero at tails. Maker fee = 0 bps.
  - Chain: Polygon mainnet (137), settlements on-chain

Deployment modes:
  PAPER       — simulated fills, mocked $50 wallet, data recording forced ON
  PENNY_LIVE  — real CLOB + real wallet, HARDCODED $1 max per trade (un-overridable)
  PRODUCTION  — all guardrails lifted, requires --confirm-production flag

VPS:
  Runs on a Linux VPS (135.181.85.32) as a systemd service (polymarket-bot.service).
  Secrets stored in /dev/shm/secrets/.env (tmpfs, never touches disk).

SECTION 3: CORE ARCHITECTURE (bot.py)
================================================================================

Entry point: `polybot run` (cli.py) → TradingBot(deployment_env).start()

The TradingBot class is the central orchestrator. It owns ~30 subsystem
instances and runs 18+ concurrent asyncio tasks.

INITIALIZATION SEQUENCE:
  1. Resolve deployment environment (PAPER/PENNY_LIVE/PRODUCTION)
  2. Create DeploymentGuard (phase-aware size caps)
  3. Create OrderExecutor (paper or live CLOB client)
  4. Create TradeStore (SQLite)
  5. Create PortfolioCorrelationEngine (Pillar 15)
  6. Create PositionManager (receives executor, store, guard, PCE, book_trackers)
  7. Create WhaleMonitor (Polygon CTF log streaming via WSS, REST fallback if no WSS URL)
  8. Create TelegramAlerter
  9. Create MarketLifecycleManager
  10. Create LatencyGuard
  11. boot_start() → setup_logging → credential gate → _run()

_run() SEQUENCE:
  1. Initialize TradeStore (SQLite schema)
  2. Discover markets → wire each market (create aggregators, detectors, books)
  3. Set wallet balance
  4. Fee model validation (live only)
  5. Create WebSocket connections (MarketWebSocketPool for trades/books + L2)
  5a. Multi-core mode: create ProcessManager → start_l2_workers() → start_pce_worker()
      Replace in-process L2 trackers with SharedBookReaderAdapters; wire VaR gate queues.
      Fallback: single-process L2WebSocket (if multicore_enabled=False).
  6. Create AdverseSelectionGuard
  7. Create BookHeartbeat
  8. Create OrderStatusPoller
  9. Create StopLossMonitor
  10. Create AdverseSelectionMonitor (maker fill tracking)
  11. Launch 19+ asyncio tasks
  12. Install signal handlers
  13. await asyncio.gather(*tasks)

MARKET WIRING (_wire_market):
  For each market, creates:
  - OHLCVAggregator (YES) + OHLCVAggregator (NO)
  - PanicDetector (wired to both aggregators)
  - CompositeSignalEvaluator (imbalance + spread compression signals)
  - RegimeDetector (SI-1, if enabled)
  - MeanReversionDrift (V3, if enabled)
  - L2OrderBook per token (with BBO callback and iceberg detector)
  - L2OrderBookAdapter (wraps L2 behind OrderbookTracker interface)
  - PCE market registration

THE THREE SIGNAL PATHS:

  Path 1: PanicDetector (primary)
    YES bar close → guards → PanicDetector.evaluate() → PanicSignal
    → EQS gate → PositionManager.open_position() → entry chaser

  Path 2: DriftSignal (V3, secondary)
    Only fires when PanicDetector is SILENT (signals are uncorrelated).
    YES bar close → drift cooldown → MeanReversionDrift.evaluate() → DriftSignal
    (DriftSignal inherits BaseSignal; no PanicSignal wrapping)
    → PositionManager.open_position() [accepts BaseSignal polymorphically]

  Path 3: RPE (Pillar 14, independent)
    YES bar close → data freshness → RPE.evaluate() → SignalResult
    → EQS gate → calibration tracking → PositionManager.open_rpe_position()
    → Fast-Strike check: |fair_value − clob_price| > 0.02 AND LatencyGuard HEALTHY
      → taker order at ask+1¢ (skips OrderChaser); else normal maker/chaser path

ORDER EXECUTION FLOW:
  Signal fires → open_position() → risk gates (6 checks) → entry price (ask-1¢)
  → EQS scoring → Kelly sizing → place BUY limit order → OrderChaser
  → POST_ONLY repricing loop → fill → place SELL at TP → exit chaser → close

  RPE Fast-Strike path (SI-7):
  RPE signal fires → fast-strike gate (divergence > 2¢ AND latency HEALTHY)
  → entry price bumped +1¢ → taker BUY order (post_only=False)
  → position.fast_strike=True → chaser skipped → fill → place SELL at TP

GRACEFUL SHUTDOWN:
  stop(): _running=False → cancel_all orders → cancel chasers → stop all
  subsystems → final stats → trade_store.clear_live_state() → close()

SECTION 6: DATA LAYER (Market Data Pipeline)

Two parallel market-data transport paths:

  A) MarketWebSocketPool (src/data/websocket_client.py)
     - Main-process trade/book multiplexer built from many MarketWebSocket
       instances behind one logical interface
     - MAX_SUBSCRIPTIONS_PER_SOCKET = 50
     - Chunks tracked asset_ids into groups of 50 and starts one physical
       WebSocket listener per chunk
     - Routes ALL sockets: trades → shared trade_queue,
       book events → shared book_queue
     - Downstream consumers (_process_trades, OHLCVAggregator,
       _process_book_updates) are unaware of sharding
     - add_assets() fills existing sockets first, then spins up a new socket
       when all existing sockets are full
     - remove_assets() unsubscribes on the owning socket via asset→socket map
     - Pool reconnect_count = sum of per-socket reconnect counters

  A1) MarketWebSocket (src/data/websocket_client.py)
     - Single physical "market" channel connection used by the pool
     - TradeEvent dataclass: timestamp, market_id, asset_id, side, price, size
     - Auto-reconnect with exponential backoff (1s to 60s) + jitter
       on a per-socket basis
     - Silence watchdog: closes WS if no messages for ws_silence_timeout_s
       (default: 120.0s; raised from 10.0s on 2026-03-09 to eliminate
       spurious reconnects on low-volume 50-market shards)
     - Bounded queues: drops OLDEST on overflow (preserves latest state)
    - WebSocket frame ceiling raised: websockets.connect(..., max_size=2**24)
      to prevent 1009 "message too big" disconnects during high-volume snapshots
     - Failure domain is isolated: if one shard disconnects, only that shard
       reconnects; sibling shards remain live

  B) L2WebSocket (src/data/l2_websocket.py)
     - Dedicated connection for L2 deltas (separated to prevent delta storms
       from starving trade processing)
     - Routes to L2OrderBook instances directly
     - Handles snapshot fetching on connect/desync
    - Uses websockets.connect(..., max_size=2**24) to absorb oversized
      initial book frames and avoid repeated 1009 disconnect loops
     - Retry with exponential backoff

L2OrderBook (src/data/l2_book.py, ~790 lines):
  CRITICAL COMPONENT — full-depth local order book reconstruction

  State machine: EMPTY → BUFFERING → SYNCED ↔ DESYNCED
  Uses SortedDict from sortedcontainers for O(log n) operations.
  Bids stored with negated keys (ascending iteration = descending price).

  Key properties:
    is_reliable — True if delta_count < 50 OR seq_gap_rate < 2%
    seq_gap_rate — rolling window of last 10,000 deltas (recoverable)
      Old: desync_total / delta_count (cumulative, NEVER recovered)
      New: gaps_in_window / window_size (transient blips age out)
    best_bid, best_ask, spread_cents, book_depth_ratio

  Recovery flow:
    Seq gap detected → DESYNCED → fire on_desync callback →
    L2WebSocket._on_book_desync() → begin_buffering() →
    REST snapshot fetch (up to 3 retries) → apply_snapshot() →
    replay buffered deltas → SYNCED

  depth_near_mid_usd(cents, max_levels):
    Iterates SortedDict keys directly — ZERO Level allocations.
    Designed for the 50ms adverse selection polling loop.

OrderbookTracker (src/data/orderbook.py):
  Simpler top-of-book tracker from WS price_change events.
  L2OrderBookAdapter wraps L2OrderBook behind this interface.
  Also exposes is_reliable (always True), seq_gap_rate (0.0), delta_count (0),
  desync_total (0) as no-op stubs so all consumer code works regardless of
  whether L2 is enabled. Plain trackers have no desync concept, so is_reliable
  is unconditionally True.

OHLCVAggregator (src/data/ohlcv.py):
  Builds 1-minute bars from trade events.

  Key outputs:
    rolling_vwap         — volume-weighted average across lookback window
    rolling_volatility   — std dev of 1-min log returns
    rolling_volatility_30m — 30-minute window for TP rescaling
    rolling_volatility_ewma — EWMA σ (RiskMetrics λ=0.94, fast-reacting)
    rolling_downside_vol_ewma — EWMA σ of downside semi-variance:
      uses min(r, 0.0) so only adverse returns contribute (Pillar 11.2)
    avg_bar_volume       — mean bar volume
    last_trade_time      — updated on EVERY trade
    current_price        — last traded price

  flush_stale_bar(now): Closes current bar if open > 60s.
    Critical for low-volume markets to prevent frozen VWAP/σ.

  _BarBuilder: internal, caps at 10,000 ticks per bar to prevent
  unbounded memory growth. On overflow, keeps most recent half.

Market Discovery (src/data/market_discovery.py):
  3-tier fetching: CLOB REST → Gamma events → Gamma markets
  MarketInfo: condition_id, question, yes/no_token_ids, volume, end_date,
    event_id, liquidity, score, accepting_orders, tags

  Filters (applied by _parse_market, in order):
    1. active=True, closed=False
    2. acceptingOrders=True
    3. negRisk=False (if reject_neg_risk config)
    4. len(tokens) == 2 (binary only)
    5. Hard-block: end_date is None → reject (reason: "missing_end_date")
       Markets with no parseable end_date are permanently excluded. The CLOB
       API sometimes returns markets with null or missing end_date fields;
       these are untradeable for a mean-reversion strategy because there is
       no time-to-resolution anchor for decay scoring.
    6. Hard-block: end_date < now() → reject (reason: "already_resolved")
       Markets whose end_date is in the past are blocked regardless of the
       API's active/closed flags, which are known to be unreliable (Tier 1
       CLOB returns stale 2023-era closed markets).
    7. days_to_resolution ≥ min_days_to_resolution (default: 3)
       (reason: "resolves_too_soon"; days_left and threshold logged)
    8. volume_num_24hr ≥ min_daily_volume_usd
    9. liquidity ≥ min_liquidity_usd (if min > 0)
    10. token_id non-empty for both YES and NO (missing_token_id rejection)

  Token extraction (_normalize_gamma_market):
    - Preferred path: native `tokens` list if present (forward-compat with
      CLOB schema: [{"token_id": ..., "outcome": ...}])
    - Fallback: clobTokenIds + outcomes JSON-encoded strings (current Gamma
      schema). Both fields JSON-parsed if str, empty list on decode error.

  Discovery tier status (as of 2026-03-12):
    Tier 1 (CLOB): broken — server ignores active/closed filters, returns
      stale 2023 closed markets. Falls through to Tier 2 every cycle.
    Tier 2 (Gamma /events): primary active path, yields ~150 markets/cycle.
    Tier 3 (Gamma /markets): dormant fallback.

  L2 Load Shedding (src/bot.py):
    Markets are ranked by daily_volume_usd descending. Only the top
    max_active_l2_markets (deploy override: 25) receive full L2OrderBook
    reconstruction + IcebergDetector wiring. All remaining markets are
    assigned lightweight OrderbookTracker instances (price/trade only,
    no depth data). This prevents L2 delta storms from overloading the
    event loop on low-volume shards that yield negligible depth alpha.
    Load shedding is logged at startup: "load_shedding_applied" with
    total_markets, l2_active count, and l2_limit.

PanicDetector (src/signals/panic_detector.py):

    2. VWAP/σ validity: σ > 0, vwap > 0

    4. Intra-bar retracement discount: if price already reverting within bar

       zscore_threshold; default 0.684 — specifically tuned for panic detection)
    7. Volume ratio gate: v_ratio ≥ volume_ratio_threshold
    8. NO discount check: no_best_ask < no_vwap × no_discount_factor

    9. Trend guard: suppress if YES has moved ≥ trend_guard_pct over

MeanReversionDrift (src/signals/drift_signal.py):

EdgeFilter / EQS (src/signals/edge_filter.py):
  PRE-TRADE QUALITY GATE — geometric mean of 4 normalised factors:

  EQS = 100 × R^0.35 × F^0.30 × T^0.20 × S^0.15

  Where:
    R = regime_quality = binary_entropy(entry_price)
        Markets near 50/50 → max uncertainty → noise → mean-revertible
    F = fee_efficiency = max(floor, 1 - fee_cents/gross_cents)
        floor = 0.30 prevents fees from dominating score on partial-fee cases
        Hard veto pre-empts: if fee_cents >= gross_cents → score=0, viable=False,
        reason="negative_ev_after_fees" (fires before geometric mean formula)
    T = tick_viability = min(1.0, discretised_net_cents / 3.0)
        How many 1¢ ticks the expected move spans on Polymarket grid
    S = signal_quality:
        Panic: 0.5 + 0.35(1-exp(-0.5·z_excess)) + 0.20·min(v_excess, 2.0)
        Model: clamp(confidence, 0.1, 1.0)
        Whale bonus: +0.15

  CRITICAL PROPERTY: geometric mean → any factor at zero zeros entire score.

  Regime-adaptive threshold (OE-6):
    When vol is available: ratio = ewma_vol / eqs_vol_ref
    adjustment = clamp(1 - ratio, -0.25, +0.25)
    threshold *= (1 + adjustment)
    → High vol lowers threshold, low vol raises it

  ConfluenceContext: whale_strong_confluence, spread_compressed, l2_reliable,
    regime_mean_revert — 4 binary factors

  compute_confluence_discount(ctx, base_threshold):
    1. HARD GATE: if !l2_reliable → return base (no discounts)
    2. Floor: maker_routing_active → 40.0, else → 35.0 (Flaw 1)
    3. Accumulate discounts for whale (4.0), spread (4.0), l2 (0.0), regime (0.0)
       Regime suppressed for drift signals (Flaw 2)
    4. Need ≥ confluence_min_factors active to apply
    5. Result: max(floor, base - sum(discounts))

RegimeDetector (src/signals/regime_detector.py):
  Per-market HMM-lite two-state model: MEAN_REVERT vs TRENDING
  Score ∈ [0, 1], higher = more mean-revertible
  is_mean_revert = score ≥ 0.40

  Three features combined via geometric mean:
    autocorrelation: lag-1 ACF mapped through sigmoid(-ac1 × 3)
      ac1 = -1 → 1.0 (strong MR), ac1 = +1 → 0.0 (trending)
    vol_ratio: ewma_vol / equal_weight_vol, sigmoid-mapped
      ratio > 2 → trending, ratio < 0.5 → calm/MR
    persistence: fraction of same-sign returns, sigmoid-mapped
      ~0.5 random → MR friendly, > 0.7 → trending

  Score smoothed via EWMA: score = λ·score + (1-λ)·raw

RPE (src/signals/resolution_probability.py):
  Pillar 14 — model-based alpha.

  CryptoPriceModel: log-normal Black-Scholes for "Will BTC exceed $X" markets
    P(S_T > K) = Φ(d2), d2 = (ln(S/K) + (0 − σ²/2)T) / (σ√T)
    Risk-neutral drift (μ=0) — prediction markets are risk-neutral
    Adaptive σ via EWMA of log-return² from spot feed

  GenericBayesianModel: Beta-Binomial model for non-crypto markets
    NOW ACTIVE — Dynamic Prior Generation Engine (DPGE) replaces static Beta(2,2)

    5-step estimate() flow:
      1. Tag prior — TagPriorRegistry.get_prior(tags) → (α₀, β₀, label)
         Falls back to prior_k market-anchored prior when dynamic priors disabled
      2. Time kernel — w_prior = sigmoid(γ × (t − t_half))
         t = days_remaining / total_duration, γ=8.0, t_half=0.15
         High early in market life, decays steeply near expiry
      3. L2 pseudo-counts — Δ = κ_eff × clamp(ln(book_depth_ratio), −2, 2)
         κ_eff = rpe_l2_kappa × w_prior; only applied when l2_reliable=True
         Buy-side pressure (ratio>1) → additive YES pseudo-counts; sell-side → NO

         w_obs = 1 − w_prior (complementary weights sum to 1)
      5. Confidence — (α_post + β_post) / (α₀ + β₀ + n_eff + |Δ| + 1) × w_prior
         Near-expiry suppression: w_prior → 0 → confidence → 0 (terminal noise filter)

  harvest_cascades() — STINK-BID CASCADE HARVESTING:
    Passive entries during panics designed to fill only on a stop-loss cascade.
    1. Reads signal.no_best_ask as the current market reference price.
    2. Calls l2_book.find_liquidity_gaps(min_depth_usd) to get thin ask levels.
    3. Filters gaps to those within tick_offset_range (default 2–4 ticks) below
       the current best ask.
    4. For each qualifying gap level, runs _check_risk_gates(), places a passive
       limit BUY via OrderExecutor at gap.price, sized at 25% of max_trade.
    5. Returns list of Position objects (id prefix "STINK-") in ENTRY_PENDING.
    Design intent: standard open_position() entries are MAKER orders placed
    near best ask. Stink bids sit 2–4 ticks deeper and fill only when retail
    stop-losses cascade through thin liquidity, acquiring the position at a
    significantly better price with zero slippage against the cascade flow.

    1. Acquire _entry_lock (serialized entries)
    2. _check_risk_gates() — returns (passed, max_trade, var_was_bisected):

       c. Max drawdown check

       g. Capital allocation

    4. Determine execution mode: maker routing + adverse selection monitor check
       If maker monitor says market not allowed → auto-downgrade to taker
    5. EQS scoring with:
       - V1 maker discount (threshold × 0.85)
       - V2 confluence discount (per active factors)
       - Regime-adaptive vol adjustment
    6. V4 probe path: if EQS fails but score ≥ probe_eqs_floor → probe entry

    11. PCE concentration haircut — SKIPPED when var_was_bisected=True
        (anti-double-penalization: bisection already reduced sizing)
    12. Deployment guard cap (PENNY_LIVE → $1)
    13. Dollar-risk cap: max_loss_per_trade_cents / (entry × 100)

StealthExecutor (src/trading/stealth_executor.py):
  SI-4 — VWAP/POV-aware time-sliced order splitting to reduce market footprint.
  Wraps OrderExecutor. Called by PositionManager._stealth for probe scale-ups.

  place_stealth_order(market_id, asset_id, side, price, total_size,
                      get_mid_fn, recent_volume_usd, book_depth_ratio):
    POV participation cap: max_slice_usd = recent_volume_usd × stealth_max_participation_pct
    Zero-volume fallback: if recent_volume_usd ≤ 0 → assume $10.00 floor
    Min-size clamp: max_slice_usd = max(max_slice_usd, stealth_min_size_usd)
    Dynamic reslicing: if ⌈notional / max_slice_usd⌉ > default slices → expand
    Emits structlog "vwap_stealth_resliced" when slice count is overridden
    Computes per-slice sizes with ±stealth_jitter_pct randomisation
    Inter-slice delays: OBI-paced via _obi_delay_ms():
      our_ratio = book_depth_ratio (BUY) or 1/book_depth_ratio (SELL)
      < 0.5 → max delay (hollowed book — wait for refill)
      > 2.0 → min delay (heavy support — execute fast)
      0.5–2.0 → random(min_ms, max_ms) (balanced — standard jitter)

  Two abandonment guards (evaluated between every pair of slices):
    1. Fill-pct guard: if already_filled / total ≥ stealth_abandon_fill_pct
       → stop slicing immediately (good-enough fill, avoid over-execution)
       Checked BEFORE inter-slice sleep.
    2. Mid-price drift guard: if get_mid_fn supplied and mid has drifted
       > stealth_abandon_drift_cents adversely since slice-1 fill
       → abandon remaining slices (market moved against us)
       Checked AFTER inter-slice sleep.

OrderChaser (src/trading/chaser.py):

Kelly Sizer (src/trading/sizer.py):
  COLD START (< min_kelly_trades = 20):
    f_cold = cold_start_frac × (min_kelly_trades - total_trades) / min_kelly_trades
    Negative-EV throttle: if rolling 10-trade expectancy < 0 → 75% reduction

  FULL KELLY:
    1. Win probability: max(0.01, decayed_wr + 0.15 × signal_score), capped at p_cap
    2. Edge discount: p_adj = 0.5 + (p_raw - 0.5) × (1 - uncertainty_penalty)
    3. Payoff ratio: b = |avg_win| / |avg_loss|
    4. Edge: p_adj × b - (1 - p_adj). If ≤ 0 → reject
    5. Kelly fraction: f* = edge / b, f_adj = f* × kelly_fraction (quarter Kelly)
    6. Capital caps: min(f_adj × bankroll, bankroll × max_pct/100, max_trade)
    7. Depth cap: further limited by available book liquidity

Fee Model (src/trading/fees.py):
  Polymarket dynamic curve: f(p) = f_max × 4 × p × (1 − p)
  Peak 2.00% at p=0.50, zero at tails. Maker = always 0 bps.

  CRITICAL FEE MODELING FIX (2026-03-12):
    Exit fees are ALWAYS modeled as taker in EQS scoring (compute_edge_score)
    and paper PnL accounting, regardless of execution_mode or market fee status.
    Reason: ~87.5% of exits cross the spread due to OrderChaser escalation,
    TP target movement, or timeout impatience. The old logic zero-ed exit fees
    in maker mode or no-fee markets, allowing guaranteed-loss trades to pass
    the negative-EV gate (fee_cents ≥ gross_cents). The bot was accepting
    3¢ spreads at p=0.50 (where round-trip fee is 4¢) and bleeding capital.
    
    Implementation: compute_edge_score() and PositionManager TP check both
    call get_fee_rate(exit_price, fee_enabled=True) unconditionally, even
    when entry is maker or the market is tagged no-fee (political category).

  compute_net_pnl_cents(entry, exit, size, *, fee_enabled,
                        is_maker_entry=False, is_maker_exit=False):
    entry_fee = 0.0 if is_maker_entry else get_fee_rate(entry)
    exit_fee  = 0.0 if is_maker_exit  else get_fee_rate(exit)
    PnL = [(exit - entry) - entry_fee - exit_fee] × size × 100

    The is_maker_entry / is_maker_exit flags (added 2026-03-12) allow
    accurate paper PnL accounting. Previously the function always applied
    the dynamic curve fee to BOTH legs regardless of execution mode, which
    over-penalised maker entries by ~2% at mid-price and under-reported
    true round-trip cost for taker exits. Callers (e.g. EdgeFilter paper
    scoring, PositionManager exit logging) now pass the actual execution
    mode so simulated PnL matches live fills.

  compute_roundtrip_fee_cents also accepts is_maker_entry / is_maker_exit
    with identical semantics (entry_fee = 0 when maker, else curve fee).

  compute_adaptive_stop_loss_cents(sl_base, entry, fee_enabled):
    trigger   = sl_stretched − fee_drag_cents
    min_floor = max(2.5, fee_drag_cents + 1.0)   (≥1¢ breathing room above fee drag)
    SL_trigger = max(min_floor, trigger)
    (at p=0.50 with fee_drag≈3¢: min_floor=4¢ — prevents chop-out at the
     old 1¢ hard floor that collapsed stops in high-fee zones)

Take Profit (src/trading/take_profit.py):
  Base model: target = entry + α × (VWAP_no - entry)

  Alpha adjustments (6 factors):
    1. Volatility (asymmetric): high → +0.03×min(vol_factor−1, 1.5), low → −0.08
       [was +0.04×(vol_factor−1) unbounded — could add +0.36 on 10× vol events]
    2. Book depth: deep → +0.03
    3. Whale confluence: +0.08
    4. Time decay: < 14d → −0.05
    5. VWAP proximity: > 10% discount → +0.04, < 2% → −0.05
    6. Clamp to [alpha_min, alpha_max]

  Dynamic fee-aware alpha_min floor (added 2026-03-12):
    At low entry prices (< 0.30) where the quadratic fee curve is steep,
    a dynamic alpha_min is computed so the TP target covers ≥ 2× round-trip
    fees. Formula:
      rt_fee = get_fee_rate(entry) + get_fee_rate(entry + 0.04)
      required_spread = 2.0 × rt_fee
      dynamic_alpha_min = required_spread / (VWAP_no − entry)
      alpha = max(alpha, dynamic_alpha_min)
      alpha = min(alpha, alpha_max)
    This prevents the bot from accepting TP targets in the fee-dead-zone
    where round-trip costs exceed the expected reversion profit.

  Fee-floor enforcement: Newton step to widen target if spread < fees

  Dynamic spread rescaling (Pillar 3): σ-proportional TP width adjustment

StopLossMonitor (src/trading/stop_loss.py):
  EVENT-DRIVEN — fires on BBO change callbacks, no polling loop.

  Fixed stop: unrealised_loss ≥ sl_trigger_cents → force_stop_loss()
  Trailing stop: activates after profit ≥ be_activation_cents
    trail_floor = hwm - trailing_offset/100
    loss = max(trail_loss, entry_loss) — uses worse of both

  V4 probe breakeven: if probe + profitable → fire on_probe_breakeven callback
    (triggers scale_probe_to_full in PositionManager)

  Pillar 11.1/11.2 — Volatility-Adaptive Baseline & Trailing (at position open):
    Both sl_trigger_cents and trailing_offset_cents are pre-computed at
    position-open time using downside semi-variance (σ_downside) instead
    of total rolling volatility.  This ensures only adverse moves widen
    the stop — a favourable price spike does not inflate the offset.

    Baseline SL (compute_adaptive_stop_loss_cents):
      multiplier = clamp(σ_downside / σ_ref, 1.0, max_mult)
      trigger   = (base_sl × multiplier) − fee_drag
      min_floor = max(2.5, fee_drag + 1.0)   (dynamic: ≥1¢ above fee drag)
      SL_trigger = max(min_floor, trigger)

    Trailing offset (compute_adaptive_trailing_offset_cents):
      multiplier = clamp(σ_downside / σ_ref, 1.0, max_mult)
      offset = base_offset × multiplier  (no fee deduction)

    StopLossMonitor reads pos.trailing_offset_cents per position,
    falling back to the global config if the position value is 0.

  Pillar 11.3 — Preemptive Liquidity Drain & Time-Decay Stop:
    Two runtime guards evaluated inside _evaluate_position():

    A. L2-Aware Preemptive Stop:
       our_support_ratio = book_depth_ratio   (bid/ask — direct, not inverted)
       [was 1/bdr which triggered on strong bids — logic inversion fixed 2026-03-10]
       If our_support_ratio < sl_preemptive_obi_threshold (0.10)
         AND unrealised_pnl < 0 (underwater):
         → force_stop_loss(reason="preemptive_liquidity_drain")
       Invariant: never fires on healthy (profitable) positions.
       Prevents slippage bleed when bid depth evaporates on exit.
       [Threshold lowered from 0.20 to 0.10 on 2026-03-10: Polymarket books
        are naturally thin with BDR routinely fluctuating 0.10-0.25; the old
        0.20 threshold caused false-positive premature exits on normal books.]
       Preemptive exits now also record a stop-loss cooldown (300s) to prevent
       the loser-loop that occurred when repeated re-entry followed preemptive
       drain exits without any inter-signal pause.

    B. Time-Decaying Vol Multiplier:
       After sl_decay_start_minutes (5 min), the extra volatility room
       given at entry decays exponentially toward 1.0×:
         decay_factor = exp(−ln(2) × (elapsed − start) / half_life)
         current_mult = 1.0 + (initial_mult − 1.0) × decay_factor
         decayed_sl = max(1.0, base_cents × current_mult − fee_drag)
       [was exp(−(elapsed−start)/half_life) — mean-lifetime, not half-life; fixed 2026-03-10]
       At half_life=15 min, a 1.5× multiplier decays to:
         T+5  → 1.50  (decay starts)
         T+20 → 1.25  (exactly 50% of extra room remaining at T+20 = start+15)
         T+35 → 1.12  (25% remaining)
         T+50 → 1.06  (12.5% remaining)

MatchingEngine (src/backtest/matching_engine.py, ~750 lines):
  ELIMINATES THREE DEADLY SINS OF NAÏVE BACKTESTING:
    1. Look-ahead bias: orders invisible until submit_time + latency_ms
    2. Queue-position optimism: maker orders track queue_ahead, fill only
       after sufficient taker volume drains the queue (FIFO)
    3. Latency blindness: configurable latency penalty (default 150ms)

  Book: SortedDict from sortedcontainers (bids negated for desc order)
  Taker execution: walks opposite side level-by-level (VWAP slippage)

SECTION 14: KEY FORMULAS & ALGORITHMS
================================================================================

EQS (Edge Quality Score):
  EQS = 100 × R^0.35 × F^0.30 × T^0.20 × S^0.15
  R = binary_entropy(p) = -p·log₂(p) - (1-p)·log₂(1-p)
  F = max(0.10, 1 - fee_cents/gross_cents)
  T = min(1.0, discretised_net_cents / 3.0)
  S = 0.5 + 0.35(1-exp(-0.5·z_excess)) + 0.20·min(v_excess, 2.0)

Kelly Criterion:
  f* = (p·b - q) / b, where b = |avg_win/avg_loss|, q = 1-p
  f_adj = f* × 0.25 (quarter Kelly)
  Pillar 15.1 — Covariance-Adjusted Kelly (MCPV):
    MCPV = Σ_i (w_i × ρ(proposed, market_i)) / Σ_i w_i
    w_i = entry_price_i × entry_size_i  (dollar exposure)
    Activation threshold: MCPV > 0.65 (below → no penalty)
    penalty_mult = max(0.05, 1 − MCPV)
    f_adj = f* × k_mult × penalty_mult
    No open positions or ρ ≤ 0.65 → penalty_mult = 1.0 (no change)
    Maximally correlated (ρ = 1) → penalty_mult = 0.05 (95% reduction)

SI-5 — Order Flow Imbalance (OFI) — Dual Role:
  Aggregate OFI = ΣΔBidQty − ΣΔAskQty  (2-second rolling window per L2OrderBook)
  Panic veto: if YES-token OFI > ofi_veto_threshold (default 50.0)
    and BUY_NO signal fires → veto (institutional momentum trap).
  Per-price wall OFI: L2OrderBook.opposing_ofi_at_price(price, side)
    Returns total volume consumed from a level over the 2-second window.
    Used by OrderChaser._should_preemptive_exit() to detect whale wall exhaustion.
  Preemptive exit trigger: consumed > ofi_wall_threshold × iceberg.estimated_total
    OR book_depth_ratio < ofi_depth_ratio_floor → cancel peg + market-sell 1 tick.

Dynamic Fee Curve:
  f(p) = f_max × 4 × p × (1 − p), f_max = 0.0200

Take-Profit:
  target = entry + α × (VWAP - entry)
  α adjusted by: vol(±0.04-0.08), depth(+0.03), whale(+0.08),
    time(-0.05), VWAP proximity(±0.04-0.05)

Dynamic Alpha (Adverse Selection):
  α_dynamic = α_base × (σ_rolling / σ_ref)^γ, clamped [0.01, 0.15]

Welch t-test (one-sided):
  t = x̄ / (s / √n), H₀: μ ≥ 0, H₁: μ < 0
  Pure-Python CDF via Lentz continued-fraction (no scipy)

VaR (Portfolio):
  VaR = z × √(w' Σ w), Σ_ij = σ_i × σ_j × ρ_ij, z = 1.645 (95%)

Regime Score:
  raw = (autocorr × vol_ratio × persistence)^(1/3)  (geometric mean)
  score = λ·score + (1-λ)·raw  (EWMA smoothing)

EWMA Volatility:
  σ²_t = λ·σ²_(t-1) + (1-λ)·r²_t, λ = 0.94 (RiskMetrics)

Volatility-Adaptive Stop-Loss (Pillar 11.1 → 11.2 → 11.3):
  Uses downside semi-variance (σ_down) instead of total EWMA vol:
    downside_r = min(r, 0)  — only adverse returns contribute
    σ²_down_t = λ · σ²_down_{t-1} + (1-λ) · downside_r²
  Baseline stop:
    multiplier = clamp(σ_down / σ_ref, 1.0, max_multiplier)
    trigger   = (base_sl × multiplier) − fee_drag_cents
    min_floor = max(2.5, fee_drag_cents + 1.0)   (≥1¢ above fee drag — dynamic)
    SL_trigger = max(min_floor, trigger)
  Trailing offset:
    multiplier = clamp(σ_down / σ_ref, 1.0, max_multiplier)
    trailing_offset = base_offset × multiplier  (no fee deduction)
  Defaults: σ_ref = 0.70, max_multiplier = 1.5
  Cold-start / missing vol / adaptive=False → multiplier = 1.0
  Pillar 11.3 — Preemptive Liquidity Drain:
    our_support_ratio = 1 / book_depth_ratio
    Trigger if our_support_ratio < 0.20 AND unrealised_pnl < 0
  Pillar 11.3 — Time-Decay Tightening:
    After sl_decay_start_minutes (5):
      decay_factor = exp(-(elapsed - start) / half_life)
      current_mult = 1.0 + (initial_mult - 1.0) × decay_factor
      decayed_sl = max(1.0, base_cents × current_mult − fee_drag)
    half_life = 15 min.  sl_vol_multiplier stored on Position at open.

Crypto RPE (Black-Scholes):
  P(S_T > K) = Φ(d₂)
  d₂ = (ln(S/K) + (0 - σ²/2)T) / (σ√T)  [risk-neutral, μ=0]

DPGE (Dynamic Prior Generation Engine) — GenericBayesianModel:
  Time-decay kernel (sigmoid):
    t = days_remaining / total_duration_days
    w_prior(t) = 1 / (1 + exp(−γ × (t − t_half))),  γ=8.0, t_half=0.15
    → near t=0 (at expiry): w_prior → 0  (prior irrelevant, suppress noise)
    → near t=1 (new market): w_prior → 1  (prior dominates sparse data)

  L2 imbalance pseudo-counts:
    raw = clamp(ln(book_depth_ratio), −2.0, +2.0)
    κ_eff = rpe_l2_kappa × w_prior
    Δ_yes = max(0, raw × κ_eff),  Δ_no = max(0, −raw × κ_eff)

  Bayesian posterior (Beta-Binomial conjugate update):
    n_eff = min(200, obs_weight × w_obs / (w_prior + ε)),  w_obs = 1 − w_prior
    α_post = α₀ + market_price × n_eff + Δ_yes
    β_post = β₀ + (1 − market_price) × n_eff + Δ_no
    probability = α_post / (α_post + β_post)

  Confidence (near-expiry suppression):
    concentration = α_post + β_post
    confidence = (concentration / (α₀ + β₀ + n_eff + Δ_total + 1)) × w_prior

SECTION 15: IMPORTANT DESIGN PATTERNS & GOTCHAS
================================================================================

1. FROZEN DATACLASS CONFIGS
   StrategyParams is frozen. Tests override via object.__setattr__ + try/finally.
   Never modify settings.strategy directly in production code.

2. FIRE-AND-FORGET PATTERN
   _safe_fire_and_forget(coro) wraps in asyncio.ensure_future with
   _safe_task_done_callback to log any unhandled exceptions.
   Used for Telegram notifications (off the hot path).

3. DUAL BOOK SYSTEMS
   L2OrderBook (full depth, SortedDict) vs OrderbookTracker (top-of-book).
   L2OrderBookAdapter bridges L2 behind Tracker interface.
   Both coexist — some code paths use L2, others use Tracker.
   Both types expose is_reliable, seq_gap_rate, delta_count, desync_total:
   plain trackers return safe non-L2 defaults (True, 0.0, 0, 0).
   Consumer code MUST NOT isinstance-check — treat both as duck-typed.

4. FAST_KILL_EVENT
   asyncio.Event shared between AdverseSelectionGuard and all OrderChasers.
   Cleared = pauses all chasers. Set = resume.
   Critical path: 50ms adverse detection → clear event → cancel orders.

5. ENTRY LOCK (asyncio.Lock)
   PositionManager._entry_lock serializes all position entries.
   Prevents per-event burst from opening multiple positions simultaneously.

6. NEGATED BIDS IN SORTEDDICT
   Both L2OrderBook and MatchingEngine store bids with negated prices:
   _bids[−price] = size. This makes SortedDict ascending iteration
   yield prices in descending order (best bid first).

7. SEQUENCE GAP TRACKING
   L2OrderBook.desync_total is NEVER reset (cumulative lifetime counter).
   desync_count resets on each snapshot.
   seq_gap_rate now uses a 10,000-delta rolling window (deque[bool]):
     _seq_gap_window tracks True (gap) / False (clean) per delta.
     _seq_gap_window_gaps is a running count of gaps in the window.
     Markets recover is_reliable=True after ~10K clean deltas.
   Old: seq_gap_rate = desync_total / delta_count (permanent damage).
   is_reliable = gap_rate < 2%. This drives the L2 hard gate decision.

8. MAKER FEE = 0
   Polymarket charges 0 bps for maker (POST_ONLY) orders.
   This is why V1 maker routing eliminates the fee_efficiency factor.
   compute_edge_score with execution_mode="maker" → entry/exit fees = 0.

   L2OrderBook.depth_near_mid_usd iterates SortedDict keys directly.
   No Level objects allocated. Designed for 50ms adverse selection loop.

10. BAR BUILDER TRUNCATION
    OHLCVAggregator._BarBuilder caps at 10,000 ticks per bar.
    On overflow, keeps most recent half but preserves _true_open and
    _total_volume. Prevents unbounded memory in high-frequency markets.

    Every 5 minutes: checkpoint_orders + checkpoint_positions to SQLite.
    On restart: restore_orders + restore_positions.

    TradeStore.get_rolling_expectancy filters WHERE is_probe=0.

14. BACKFILLED TICK TIMESTAMPS CAN SILENTLY KILL REPLAY
  Historical Polymarket backfills may store all trades in a day file with
  local_ts values near the backfill execution time rather than the file's
  calendar date. If those compressed timestamps are replayed verbatim,
  OHLCVAggregator may never accumulate a full 60-second span per asset,
  causing zero bar closures and therefore zero signal evaluations.
  Symptom: WFO runs for hours or days with 0 fills, 0 PnL, and no explicit
  error. The fix lives in DataLoader and should be preserved on both JSONL
  and Parquet paths.

15. ZERO-TRADE WFO IS OFTEN A DATA PROBLEM FIRST, NOT AN ALPHA PROBLEM
  When total_positions=0 across all folds, check bar formation before tuning
  thresholds. In the March 8 case, the strategy was not "too strict" at
  first; the adapter never reached PanicDetector because no bars closed.
  After repair, the real blocking gate was visible: zscore_low.

16. PARALLEL OPTUNA WFO MUST KEEP THE MANUAL ASK/TELL WORKER LOOP
  With max_workers > 1 and warm_start=True, multiple workers can race on the
  same enqueued seed trial. Using study.optimize() directly in each child
  process can surface Optuna's "Cannot tell a COMPLETE trial" error.
  The current worker implementation avoids this by using study.ask()/tell(),
  ignoring the duplicate COMPLETE-trial race, and marking unexpected failures
  as failed trials so they do not remain RUNNING in SQLite.

19. EXCEPTION CIRCUIT BREAKER (src/core/exception_circuit_breaker.py)
    Critical async loops (_process_trades, AdverseSelectionGuard._decision_loop)
    MUST NOT use blanket `except Exception: pass/continue`.  Instead:
    a. Targeted catches for expected, non-fatal exceptions (KeyError on stale
       data, AttributeError on missing books, asyncio.TimeoutError).
    b. Remaining Exception → log error + ExceptionCircuitBreaker.record().
    c. If the breaker trips (5 errors in 60s rolling window) → CRITICAL log,
       Telegram alert, and immediate graceful shutdown (TradingBot.stop()).
    Silent exception swallowing is FORBIDDEN in hot-path loops.  Every new
    loop must instantiate its own ExceptionCircuitBreaker and wire the
    shutdown path.  The breaker uses time.monotonic() with a deque of
    timestamps; once tripped it latches (always returns True).

20. VARIABLE SCOPE IN CONDITIONAL SIGNAL PATHS (UnboundLocalError trap)
    Variables assigned INSIDE `if sig:` (panic fires) that are also read in
    the `if not sig:` (drift fires) branch cause UnboundLocalError when only
    a drift signal arrives.  The `_regime_score` variable in
    `_on_yes_bar_closed` was an instance of this: assigned only when
    PanicDetector returned a signal, but referenced on the drift code path.
    The silent failure propagated into `_stale_bar_flush_loop` because that
    loop calls `_on_yes_bar_closed` inside a broad except clause → crash is
    swallowed, loop continues, zero signal evaluations ever occur despite
    bars closing normally.
    FIX PATTERN: assign a safe default (e.g. `_regime_score = 0.5`) BEFORE
    the conditional, then override inside each branch as needed.
    Verified fix on March 9 VPS audit: stale_bar_flush_error dropped from
    27 events (old bot) to 0 after restart with the patch applied.

21. SHADOW MODE ROUTING & FUTURE ALPHAS
    Polymorphic Routing: The PositionManager must never contain hardcoded
    strategy names.  Routing is determined entirely by the signal's fields,
    not by ``isinstance()`` checks or string-matching on signal class names.

    The ``is_shadow`` Switch: Signals are routed to the Shadow Performance
    Tracker (SPT) pipeline strictly by checking ``signal.is_shadow is True``.
    When a signal carries ``is_shadow=True``, the PositionManager logs it
    to the SPT database (keyed by ``signal.signal_source``) and skips
    live order placement.  When ``is_shadow=False``, the signal flows
    through the normal execution path unchanged.

    Future-Proofing: Any new strategy simply inherits from BaseSignal (or
    includes ``is_shadow`` and ``signal_source`` fields for standalone
    dataclasses), sets ``is_shadow = True`` via its config flag, and
    populates ``signal_source`` with a unique identifier.  The SPT handles
    the rest automatically — shadow P&L tracking, Sharpe computation, and
    promotion readiness gating all key off ``signal_source`` with zero
    changes to the PositionManager.

SI-9 — MUTUALLY EXCLUSIVE COMBINATORIAL ARBITRAGE (DISABLED BY DEFAULT):
  Dutch Book arbitrage across negRisk (mutually exclusive) Polymarket markets.
  When the sum of all YES best-bids across an event's legs falls below 1.0 − margin,
  buying YES on every leg locks in a riskless profit equal to 1 − Σ(prices) − fees.
  Disabled by default (si9_arb_enabled=False). All negRisk market discovery is
  also gated by this flag.

  Alpha source:
    negRisk markets on the same event_id are mutually exclusive (exactly one YES
    resolves to $1, all others to $0). Dutch Book: if Σ(YES best_bids) < 1.0 then
    buying S shares of every YES token at those prices guarantees a net positive
    payoff regardless of which leg resolves.

  Architecture:
    ArbitrageClusterManager (src/data/arb_clusters.py)
      Groups MarketInfo objects by event_id into ArbCluster dataclasses.
      update_cluster(market_info): upserts a market into its cluster.
      get_clusters(): returns all clusters with ≥ 2 legs.
      Each ArbCluster holds: event_id, List[MarketInfo], best_bid per leg.

    ComboArbDetector (src/signals/combinatorial_arb.py)
      evaluate(cluster, wallet_balance) → Optional[ComboArbSignal]
      Gates: si9_arb_enabled, min_legs ≥ 2, max_legs ≤ si9_max_legs,
             min depth per leg ≥ si9_min_leg_depth_usd,
             Σ(best_bids) < 1.0 − si9_min_margin_cents/100.
      Delegates sizing to ComboSizer.

    ComboSizer (src/signals/combinatorial_arb.py)
      Share-count-pegged sizing: S = min(budget, si9_max_per_combo_usd) / Σ(prices)
      All N legs receive identical share count S — the fundamental Dutch Book
      invariant. NOT Kelly / NOT USD-uniform across legs.
      Respects si9_max_per_combo_usd and si9_max_total_exposure_usd.
      wallet_balance passed in from bot._combo_arbitrage_loop() so the sizer
      can enforce si9_max_total_exposure_usd against real available capital.

    ComboLeg / ComboArbSignal (src/signals/combinatorial_arb.py)
      ComboLeg: market_id, yes_asset_id, price, size (shares)
      ComboArbSignal: event_id, legs: List[ComboLeg], expected_profit_cents,
        margin_cents, total_cost_usd

  Execution — bot._combo_arbitrage_loop():
    Polls ArbitrageClusterManager every si9_scan_interval_ms.
    On ComboArbSignal: calls PositionManager.open_combo_position(signal).
    Guarded by _combo_breaker (threshold=5, window_s=60s).
    Breaker reset in _suspend_and_reset().

  Hanging Leg State Machine — PositionManager.abandon_combo():
    Called when any leg remains unfilled beyond si9_max_leg_delay_ms.
    State 1 (zero fills across all legs):
      Safe cancel — cancel all resting maker orders; no net exposure.
    State 2 (≥1 filled leg):
      Attempt emergency hedge: cross the spread on each unfilled leg
      (taker), up to si9_emergency_taker_max_cents above best_ask.
      If spread too wide (> si9_emergency_max_spread_cents): dump all
      filled legs via taker sell (accept loss to close net exposure).
    Position transitions: ENTRY_PENDING → ACTIVE (full fill) or
      ABANDONED (partial/zero fill after abandon).

  PCE integration — combo_positions_for_pce():
    Returns each combo leg as a full _MinimalPosition-compatible dict
    (NOT tuples) with keys: market_id, entry_price, size, filled_size,
    trade_asset_id, no_asset_id, event_id. Included in _check_var_remote()

March 11th — CRITICAL: Ghost Liquidity Crash-Loop Fix (3 bugs):
  Problem: Bot was trapped in a continuous crash-and-restart loop.  The
  external watchdog / systemd service kept triggering cold boots evidenced by
  recurring "Bot starting in PAPER mode" log entries.  Every restart was
  preceded by a cluster of 5-6 ghost_liquidity_detected events, perfectly
  aligning with the _ghost_breaker threshold of 5 errors per 60 seconds.

  Root-cause analysis identified three cascading bugs in src/bot.py:

  Bug 1 — Fatal: _suspend_and_reset() killed the process (CRITICAL)
    _suspend_and_reset() called task.cancel() on the WS task (line ~847).
    That task is part of asyncio.gather(*self._tasks, return_exceptions=False)
    in start().  When a gather-managed task is cancelled, gather propagates
    the CancelledError upward.  start() caught it, exited the try block,
    returned — event loop tore down — process died — watchdog restarted.
    This is the primary crash vector.  All circuit-breaker handlers in all 6
    hot-path loops trigger _suspend_and_reset(), so any breaker trip produced
    a guaranteed kill.

  Bug 2 — Unprotected breaker-trip handlers
    All 6 except-Exception blocks called `await self._suspend_and_reset()`
    with no wrapper.  If _suspend_and_reset() itself raised (e.g. a network
    error during ws.stop()), the exception escaped the except handler, killed
    the async task, and crashed the process through gather.

  Bug 3 — Unprotected force-exit calls in ghost loop
    Inside the emergency drain path, force_stop_loss() and cancel_order()
    were called without try/except.  These are network/executor calls that
    can fail (API errors, timeouts), producing raw exceptions that accumulated
    against the _ghost_breaker threshold.  Within 60 seconds of sustained
    ghost detections, 5 executor errors tripped the breaker → called
    _suspend_and_reset() → killed the process (Bug 1).

  Crash sequence:
    1. Force-exit calls during ghost drain throw executor exceptions (Bug 3)
    2. Each exception caught → _ghost_breaker.record() increments
    3. After 5th error, breaker trips → calls _suspend_and_reset()
    4. _suspend_and_reset() cancels WS task inside gather (Bug 1)
    5. gather propagates CancelledError → start() exits → process dies
    6. Watchdog restarts → "Bot starting in PAPER mode"

  Fixes applied to src/bot.py:

  Fix 1 — Remove task.cancel() from _suspend_and_reset():
    Replaced explicit `task.cancel()` with `ws.stop()` + yield to event loop
    (`await asyncio.sleep(0)`) + wait for natural task completion via
    `asyncio.wait_for(asyncio.shield(task), timeout=5)`.
    The WS task is no longer forcibly cancelled, so gather never receives an
    unsolicted CancelledError from within _suspend_and_reset().

  Fix 2 — Wrap entire _suspend_and_reset() in try/except:
    Added outer try/except(Exception) around the full body of
    _suspend_and_reset() so any internal failure logs via log.error() and
    returns cleanly.  CancelledError is explicitly re-raised to preserve
    correct task cancellation semantics.  Updated docstring to document the
    "must never raise" contract.

  Fix 3 — Wrap per-position force-exit calls in ghost loop:
    Each call to force_stop_loss() / cancel_order() during the emergency
    drain position-force-exit block is now wrapped in try/except(Exception)
    with a log.warning("ghost_force_exit_error").  A failed executor call for
    one position no longer propagates and feeds the circuit breaker.

  Fix 4 — Wrap cancel_vacuum_stink_bids() in recovery section:
